# 02: Teaching a Neural Network to Learn

In Project 1 we built a single neuron that decided whether to bring an umbrella. It worked, but the weights were hand-picked. We chose 0.6 for cloud cover and 0.3 for humidity because they seemed reasonable. The neuron had no say in it.

That is not a network that learns. That is a human doing the learning and writing the result into a script.

This project changes that. The network picks its own weights from examples. We give it labelled data and it figures out what the weights should be entirely by itself.

---

## What is new in this project

Three things are new here that were not in Project 1.

**A network of neurons arranged in layers.** Instead of one neuron, we now have three layers of neurons. The hidden layer in the middle is where the network learns to combine the input features in ways we would never think to specify by hand.

**A loss function.** The network needs a way to measure how wrong it is at any given moment. That measurement is called the loss. Without it, the network has no signal to learn from.

**Backpropagation.** The network uses the loss measurement to adjust its weights in the right direction. This process is called backpropagation and it is the core mechanism behind how every neural network learns, from the smallest toy example to the largest language model.

---

## The training data

We are still solving the umbrella problem. Here are six examples of weather conditions, each labelled with the correct answer:

```text
cloud_cover  humidity  wind_speed    bring_umbrella
    0.9         0.7       0.3              1  (yes, bring it)
    0.1         0.2       0.1              0  (no, leave it)
    0.8         0.6       0.5              1  (yes)
    0.2         0.3       0.2              0  (no)
    0.7         0.8       0.4              1  (yes)
    0.1         0.1       0.9              0  (no)
```

Six examples is a tiny dataset. A real network would use thousands. But six is enough to see the learning happen clearly and to follow every calculation by hand.

---

## The network architecture

The network has three layers connected in sequence.

**Input layer (3 neurons):** One neuron per feature: cloud cover, humidity, and wind speed. This layer does no computation. It just holds the input values and passes them forward.

**Hidden layer (4 neurons):** Each neuron in this layer receives all three inputs, multiplies them by its own weights, adds a bias, and passes the result through a sigmoid activation. This is where the interesting learning happens. The hidden layer can learn to detect combinations of features that the raw inputs alone do not capture.

**Output layer (1 neuron):** Receives the four outputs from the hidden layer and produces a single number between 0 and 1: the predicted probability of needing an umbrella.

In matrix terms, the shapes at each step are:

```text
training_inputs:          shape (6, 3)   6 examples, 3 features each
weights_input_to_hidden:  shape (3, 4)   connects 3 inputs to 4 hidden neurons
bias_hidden:              shape (1, 4)   one bias per hidden neuron
hidden_layer_output:      shape (6, 4)   4 outputs for each of the 6 examples
weights_hidden_to_output: shape (4, 1)   connects 4 hidden neurons to 1 output
bias_output:              shape (1, 1)   one bias for the output neuron
predicted_output:         shape (6, 1)   one prediction per example
```

At the start, both weight matrices are filled with small random numbers. The network knows nothing yet.

---

## Why 4 hidden neurons?

There is no formula for this. The number of hidden neurons is guided by intuition and trial and error. A few rough guidelines:

**Stay between the input size and the output size.** Our input is 3 features and our output is 1 number, so anywhere from 2 to 8 hidden neurons is a reasonable starting range for a problem this simple. 4 sits comfortably in that range.

**Too few means the network cannot learn the pattern.** This is called underfitting: the network is not powerful enough to represent the decision boundary.

**Too many risks memorising the training data.** This is called overfitting: the network learns the specific examples rather than the general rule, and performs poorly on new data it has not seen before. With only 6 training examples, even 4 hidden neurons is already capable of overfitting. It does not matter here because we are only illustrating the concept, not deploying the network.

**Powers of 2 are a practical convention.** Real networks typically use 32, 64, 128, or 256 hidden neurons per layer. Modern hardware is optimised for matrix operations on these sizes, so they tend to train faster. This is a practical habit, not a mathematical rule.

---

## The forward pass

The forward pass is the same calculation as Project 1, applied twice in sequence.

### Layer 1: input to hidden

```text
hidden_layer_input  = training_inputs . weights_input_to_hidden + bias_hidden
hidden_layer_output = sigmoid(hidden_layer_input)
```

The dot product multiplies each input example by the weight matrix, adds the bias, and sigmoid squashes every value into the range (0, 1). The result is a (6, 4) matrix: four hidden neuron outputs for each of the six training examples.

### Layer 2: hidden to output

```text
output_layer_input = hidden_layer_output . weights_hidden_to_output + bias_output
predicted_output   = sigmoid(output_layer_input)
```

The four hidden outputs feed through the second weight matrix and sigmoid to produce the final (6, 1) output: one predicted probability per example.

At the very start, with random weights, these predictions are essentially random. Maybe 0.48, 0.51, 0.53. The network is guessing. The job of training is to push those predictions toward the correct labels.

---

## The loss function

To teach the network anything, we need a number that tells it how wrong it is. That number is the loss.

For this project we use **mean squared error (MSE)**:

```text
loss = mean( (actual - predicted)^2 )
```

For each training example, take the difference between the correct label and the predicted output, square it (so the sign does not matter), and average across all examples. If all predictions are perfect, the loss is 0. If the predictions are terrible, the loss is large.

On the first pass with random weights:

```text
initial loss = 0.3271
```

Every training step from here is aimed at reducing this number. After 5000 training steps it reaches 0.0004.

---

## Backpropagation: the intuition

Before the math, here is the idea in plain terms.

Imagine the loss as the height of a hill. The network's weights are its position on that hill. The goal is to walk downhill until reaching the lowest point, which corresponds to the minimum possible loss.

To walk downhill, you need to know which direction is downhill. The **derivative** of the loss with respect to each weight gives exactly that information. If increasing a particular weight causes the loss to rise, decrease that weight. If increasing it causes the loss to fall, increase it. This process of repeatedly stepping in the direction that reduces the loss is called **gradient descent**.

**Backpropagation** is the algorithm that computes all those derivatives efficiently. It works by applying the chain rule of calculus backwards through every layer, starting from the output and working back to the inputs.

---

## Backpropagation: the math

The key identity that makes this practical is the derivative of the sigmoid function.

If the sigmoid output is `s`, then:

```text
sigmoid'(x) = s * (1 - s)
```

This is elegant because you already computed `s` on the forward pass. You do not need to recompute anything. You just use the values you already have.

When `s` is close to 0.5, the derivative is 0.25: the sigmoid is sensitive to small changes in its input. When `s` is close to 0 or 1 (the edges of the sigmoid curve), the derivative is nearly 0: the sigmoid is saturated and almost unresponsive. This saturation effect becomes important in very deep networks.

Now the chain rule, applied backwards through the network.

**Step 1: how much does the loss change when the predicted output changes?**

```text
gradient_loss_wrt_predicted = -2 * (training_labels - predicted_output) / n
```

This is just the derivative of mean squared error with respect to the prediction. `n` is the number of training examples.

**Step 2: how much does the predicted output change when the output layer's pre-activation input changes?**

```text
gradient_predicted_wrt_output_input = predicted_output * (1 - predicted_output)
```

This is the sigmoid derivative evaluated at the output layer.

**Step 3: combine into one gradient at the output layer.**

```text
gradient_output_delta = gradient_loss_wrt_predicted * gradient_predicted_wrt_output_input
```

**Step 4: use the output gradient to compute weight gradients for the output layer.**

The inputs to the output layer are the hidden layer outputs, so:

```text
gradient_weights_hidden_to_output = hidden_layer_output.T . gradient_output_delta
gradient_bias_output              = sum(gradient_output_delta)
```

**Step 5: carry the gradient back through the hidden layer.**

```text
gradient_loss_wrt_hidden_output  = gradient_output_delta . weights_hidden_to_output.T
gradient_hidden_wrt_hidden_input = sigmoid'(hidden_layer_input)
gradient_hidden_delta            = gradient_loss_wrt_hidden_output * gradient_hidden_wrt_hidden_input

gradient_weights_input_to_hidden = training_inputs.T . gradient_hidden_delta
gradient_bias_hidden             = sum(gradient_hidden_delta)
```

The chain rule has now carried the loss signal from the output all the way back to the first weight matrix.

**Step 6: update every weight.**

```text
weights_input_to_hidden  -= learning_rate * gradient_weights_input_to_hidden
bias_hidden              -= learning_rate * gradient_bias_hidden
weights_hidden_to_output -= learning_rate * gradient_weights_hidden_to_output
bias_output              -= learning_rate * gradient_bias_output
```

The learning rate (0.5 here) controls how large each step is. Too large and the network overshoots the minimum and oscillates. Too small and training takes a very long time to converge. Choosing the learning rate is one of the key decisions in training a real network.

This entire sequence (forward pass, compute loss, backward pass, update weights) is one training **epoch**. We repeat it thousands of times.

---

## The code

```python
import numpy as np

def sigmoid(x):
    return 1 / (1 + np.exp(-x))

def sigmoid_derivative(x):
    sigmoid_output = sigmoid(x)
    return sigmoid_output * (1 - sigmoid_output)

training_inputs = np.array([
    [0.9, 0.7, 0.3],
    [0.1, 0.2, 0.1],
    [0.8, 0.6, 0.5],
    [0.2, 0.3, 0.2],
    [0.7, 0.8, 0.4],
    [0.1, 0.1, 0.9],
])
training_labels = np.array([[1], [0], [1], [0], [1], [0]])

np.random.seed(42)
weights_input_to_hidden  = np.random.randn(3, 4) * 0.5
bias_hidden              = np.zeros((1, 4))
weights_hidden_to_output = np.random.randn(4, 1) * 0.5
bias_output              = np.zeros((1, 1))

learning_rate = 0.5

for epoch in range(5000):

    # Forward pass
    hidden_layer_input  = np.dot(training_inputs, weights_input_to_hidden) + bias_hidden
    hidden_layer_output = sigmoid(hidden_layer_input)
    output_layer_input  = np.dot(hidden_layer_output, weights_hidden_to_output) + bias_output
    predicted_output    = sigmoid(output_layer_input)

    # Loss
    loss = np.mean((training_labels - predicted_output) ** 2)

    # Backward pass
    gradient_loss_wrt_predicted         = -2 * (training_labels - predicted_output) / len(training_inputs)
    gradient_predicted_wrt_output_input = sigmoid_derivative(output_layer_input)
    gradient_output_delta               = gradient_loss_wrt_predicted * gradient_predicted_wrt_output_input

    gradient_weights_hidden_to_output = np.dot(hidden_layer_output.T, gradient_output_delta)
    gradient_bias_output              = np.sum(gradient_output_delta, axis=0, keepdims=True)

    gradient_loss_wrt_hidden_output  = np.dot(gradient_output_delta, weights_hidden_to_output.T)
    gradient_hidden_wrt_hidden_input = sigmoid_derivative(hidden_layer_input)
    gradient_hidden_delta            = gradient_loss_wrt_hidden_output * gradient_hidden_wrt_hidden_input

    gradient_weights_input_to_hidden = np.dot(training_inputs.T, gradient_hidden_delta)
    gradient_bias_hidden             = np.sum(gradient_hidden_delta, axis=0, keepdims=True)

    # Update weights
    weights_input_to_hidden  -= learning_rate * gradient_weights_input_to_hidden
    bias_hidden              -= learning_rate * gradient_bias_hidden
    weights_hidden_to_output -= learning_rate * gradient_weights_hidden_to_output
    bias_output              -= learning_rate * gradient_bias_output

    if epoch % 100 == 0:
        print(f"Epoch {epoch:5d}  loss: {loss:.4f}")
```

### Experiment 2: 5000 epochs

Both experiments get every prediction right. The 800-epoch run is less confident (0.9560 on sample 1 vs 0.9895), but the decisions are the same. For this simple problem, 800 epochs is already enough.

The extra 4200 epochs reduced the loss from 0.0045 to 0.0004. Whether that matters depends entirely on the problem. For a six-example toy dataset, it does not. For a production model predicting something with real consequences, the difference between 0.0045 and 0.0004 could matter a great deal.

## The shape of learning: diminishing returns

Looking at both experiments reveals a pattern that appears in every neural network, not just this one.

The loss drops from 0.3271 to 0.0075 in the first 600 epochs. That is a 97.7% reduction. The network does the vast majority of its learning almost immediately.

The remaining 4400 epochs (600 to 5000) only reduce the loss by another 0.007: the remaining 2.3%.

```text
Epochs 0 to 600:     97.7% of total learning
Epochs 600 to 5000:   2.3% of total learning
```

This is called **diminishing returns** in gradient descent. The first steps downhill are large because the network is far from the minimum and the slope is steep. As it gets closer to the minimum, the slope flattens and each step buys less improvement.

The practical implication: training ten times as long does not produce ten times better results. At some point, extra training stops being useful. Knowing when to stop is a judgment call that comes with experience and depends on whether the remaining loss matters for your specific use case.

## What just happened

At the start, the weights were random numbers. The network was guessing.

After 5000 epochs of forward pass, loss computation, backpropagation, and weight updates, the network found a set of weights that correctly separates umbrella days from non-umbrella days.

Nobody told it that cloud cover matters more than wind speed. Nobody told it to pay attention to humidity on overcast days. It discovered all of that from six labelled examples and five thousand small adjustments.
